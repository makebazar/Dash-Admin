# Геймификация DashAdmin: Система промо-акций для гостей

## 1. Общий концепт
Система предназначена для повышения лояльности гостей компьютерного клуба и стимулирования пополнений баланса/покупок в баре. Гости зарабатывают **XP (Опыт)**, повышают **Уровни** и выполняют **Квесты**, чтобы получить доступ к более ценным призам в мини-играх.

**Особенность архитектуры (V2):** Аккаунты игроков глобальны (по номеру телефона), но балансы (XP, бонусы, билеты, квесты) уникальны для каждого клуба.

---

## 2. Уровни и Прогрессия (Leveling System)

### 2.1. Механика XP
Игрок получает Опыт (XP) за следующие действия:
*   **Покупки в баре:** Автоматически при закрытии чека (напр. 10 XP за каждые 100₽).
*   **Выполнение квестов:** Основной источник XP (фиксированные награды).
*   **Пополнения баланса:** (опционально) по правилам начисления.

### 2.2. Настройка уровней (Roadmap)
Администратор настраивает таблицу порогов XP (напр. Lvl 2 = 500 XP, Lvl 3 = 1500 XP).
С ростом уровня в мини-играх (`wheel`, `dice`, `safe`) разблокируются более ценные призы.

### 2.3. Фильтрация призов по уровням
В каждой игре список доступных призов формируется динамически:
`SELECT * FROM promo_prizes WHERE game_slug = :game AND min_level <= :player_level AND max_level >= :player_level`

## 3. Система Квестов (Quests System)

Квесты — это задания, которые приносят игроку **XP, Билеты и Бонусные рубли**.

### 3.1. Типы трекинга
1.  **Авто-квесты на товары (POS):** Анализируют состав чека (`shift_receipt_items`) при привязке `promo_player_id`. 
    *   *Пример:* "Купи 2 Red Bull" (проверка количества товара в чеке).
2.  **Квесты на услуги (Manual):** Срабатывают при нажатии админом кнопок начисления билетов за услуги (пакеты времени).
    *   *Пример:* "Купи пакет 'Ночь' 3 раза".
3.  **Квесты на сумму:** "Чек от 1000₽ в баре".

### 3.2. Квесты при входе (Check-in Quests)
При чекине (сканировании QR) игроку может быть назначен случайный квест из пула `is_randomizable` с ограниченным временем действия (`lifetime_minutes`).

## 4. UI/UX в приложении игрока (PWA)

### 4.1. Header (Шапка)
*   Отображение текущего уровня игрока: **`[ LVL 5 ]`**.

### 4.2. Личный кабинет / Главная
*   **XP Progress Bar:** Полоса прогресса под балансом билетов (напр. `1250 / 2000 XP`).
*   **Действие:** При клике на уровень или Progress Bar открывается страница **«Твой Путь»**.

### 4.3. Страница «Твой Путь» (Roadmap)
*   Интерактивный список уровней.
*   Описание разблокируемых преимуществ (напр. "На 6 уровне в Сейфе появляются Энергетики").
*   Информация о том, как заработать XP.

### 4.4. Страница «Квесты»
*   **Доступ:** Отдельная иконка в нижнем навигационном меню.
*   **Контент:** Список активных квестов с индикаторами прогресса и кнопками забора наград (если квест выполнен).
*   **Таймеры:** Для временных квестов отображается обратный отсчет.

### 4.5. Нижнее меню (Navigation)
1.  **Главная** (Игры)
2.  **Квесты** (Задания)
3.  **Кошелек** (Призы)
4.  **Профиль** (История)

## 5. Техническая архитектура

### Основные таблицы:
*   `promo_levels`: Настройка порогов XP для уровней.
*   `promo_quests`: Конфигурация квестов (триггеры, награды, лимиты).
*   `promo_player_quests`: Состояние выполнения квестов конкретным игроком.
*   `promo_prizes`: Расширена полями `min_level`, `max_level` и `game_slug`.
*   `promo_player_balances`: Хранит `total_xp` и `bonus_balance` игрока в клубе.

### Интеграция с POS:
В момент `INSERT INTO shift_receipts` с указанным `promo_player_id`, вызывается обработчик, который:
1.  Начисляет XP за сумму чека.
2.  Обновляет прогресс квестов на основе товаров (`shift_receipt_items`).
3.  При выполнении условий квеста — шлет SSE-уведомление игроку.
