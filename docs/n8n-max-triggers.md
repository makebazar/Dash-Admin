# Триггеры (события) ноды n8n-nodes-max

Этот документ основан на анализе доступных триггеров в n8n для мессенджера Max и определяет ключевые события для реализации нашего AI-агента.

## Полный список триггеров

Нода предоставляет 11 типов событий, на которые может реагировать workflow в n8n:

1.  **On bot added to chat** - Срабатывает, когда бота добавляют в групповой чат.
2.  **On bot removed from chat** - Срабатывает, когда бота удаляют из группового чата.
3.  **On bot started** - Срабатывает, когда пользователь впервые открывает диалог с ботом.
4.  **On button clicked** - Срабатывает, когда пользователь нажимает на inline-кнопку, отправленную ботом.
5.  **On chat title changed** - Срабатывает при изменении названия группового чата.
6.  **On message deleted** - Срабатывает, когда сообщение удаляется.
7.  **On message edited** - Срабатывает, когда сообщение редактируется.
8.  **On message received chat** - Срабатывает при получении нового сообщения в групповом чате, где состоит бот.
9.  **On message received direct** - Срабатывает при получении нового личного сообщения от пользователя.
10. **On user joined chat** - Срабатывает, когда новый пользователь присоединяется к групповому чату.
11. **On user left chat** - Срабатывает, когда пользователь покидает групповой чат.

## Ключевые триггеры для нашего проекта

Для реализации задуманной нами архитектуры с привязкой аккаунта и контекстом клуба, нам потребуются в первую очередь три из этих триггеров:

### 1. `On message received direct`

Это **основной** триггер для всей нашей логики.

-   **Сценарий привязки:** Он будет ловить сообщение с 6-значным кодом от пользователя.
-   **Сценарий запроса данных:** Он будет ловить все аналитические запросы пользователя, такие как "какая выручка за вчера?" или "сколько часов отработал иван".

### 2. `On button clicked`

Это **ключевой** триггер для реализации контекста при работе с несколькими клубами.

-   **Сценарий выбора клуба:** Когда у пользователя несколько клубов, бот отправит ему сообщение с кнопками (названиями клубов). Нажатие на одну из этих кнопок вызовет этот триггер. Workflow, который будет запущен этим триггером, вызовет наш `POST /api/bot/context`, чтобы сохранить выбор пользователя.

### 3. `On bot started`

Этот триггер полезен для улучшения пользовательского опыта (User Experience).

-   **Сценарий приветствия:** Когда пользователь впервые начинает диалог с ботом, мы можем автоматически отправить ему приветственное сообщение и краткую инструкцию о том, как привязать свой аккаунт DashAdmin для начала работы.
